Indian Financial Year

  • FY runs 1 April → 31 March
  • Label example: 1 Apr 2026 – 31 Mar 2027 = 2026-27
  • Used on dashboard MIS totals, MIS Report, DIR-3 next-filing rules

Client Master — Approval

StatusMeaning
PendingNot usable in MIS, Mapping, or DIR-3
ApprovedFull use everywhere
  • New client by staff → pending until approver accepts
  • Edit approved client by staff → pending again
  • Superuser → approved immediately

Client Master — Key rules

  • Director + DIN: only Individual / Foreign Citizen; not Branch
  • PAN, name, and client type must be consistent (system validates)
  • Auto Client ID (e.g. AN0001) by name + branch
  • Delete blocked if MIS, mapping, or DIR-3 still linked
  • Bulk import: preview first; imports may be pending until approved

Director Mapping

  • Director: approved Individual/Foreign Citizen, Is Director, with DIN
  • Company: Pvt/Public Ltd, LLP, Nidhi, FPO, Sec 8 Co
  • Only one active appointment per director–company (end old before new)
  • Cessation needs a reason; dates must be logical

MIS — Who & what

  • Only approved clients (from suggestions)
  • Branch-restricted users: their branch only
  • Fees + GST = Total (auto-calculated)
  • No negative amounts; no GST when fees = 0
  • Tender: enter fees and/or deposit (non-negative)

MIS — Dashboard FY card

  • Period: 1 April (current FY) → today
  • Clients: approved only
  • Totals: fees (incl. GST), receipts, expenses
  • Tender is not in this dashboard card (use MIS Report for tender)
  • Same approved-client rule as Reports → MIS Report

MIS — Bulk upload

  • CLIENT_ID must exist, be approved, name must match file
  • Same GST rule: no GST if fees are zero/blank
  • Preview → confirm (all-or-nothing)
  • Your permissions control which types you can add (fees/receipts/expenses)

DIR-3 eKYC — Cadence

Based on financial year of last filing, not calendar years from the date:

Filing anywhere in FY 2026-27
→ next filing allowed from 1 April 2029 (FY 2029-30)

System blocks earlier dates and shows the earliest allowed date.

DIR-3 — Dashboard “pending”

  • Never filed, or
  • Today ≥ next allowed date from last filing
  • Only approved directors (with DIN, director flag, eligible type)

Reports — Overview

  • MIS figures: approved clients only
  • Branch filter follows your access
  • CSV export needs separate export permission
  • MIS Report: transactions, client-wise, type-wise, month-wise (from FY 2026-27)
  • Director report: active appointments as on a date

Permissions & branches

  • Superuser: full access
  • Others: menus/actions per assigned permissions (access groups)
  • Employee branch access: blank = all; else Trivandrum or Nagercoil only
  • Activity log: superusers only

User accounts

  • Inactive users cannot sign in
  • Cannot deactivate/delete yourself or superusers
  • May be forced to change password on first login
  • Client-user email must match one Client Master record
